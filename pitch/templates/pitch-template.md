## 작성 가이드

### Background 작성 팁
- 간결하고 명확하게 (3-5문장)
- 비즈니스 임팩트나 기술적 필요성 강조
- 데이터나 수치가 있다면 포함

### Solution 작성 팁
- **핵심 가치**: 표 형태로 간단명료하게
- **차별화 기능**:
  - 문제 인식: 현재 상황의 문제점
  - 해결 방식: 구체적인 해결 방법
  - 효과: 측정 가능한 결과

### User Story 작성 팁
- `[카테고리]`: 기능 영역 (예: 포트폴리오, 지갑, 설정 등)
- 사용자 관점에서 기능 단위로 작성
- Test Case는 실제 UI 상호작용을 상상할 수 있게
- "사용자는 ~을 할 수 있다" 형태로 작성
- 각 Test Case는 하나의 명확한 동작만 설명

### Test Case 작성 예시
- ✅ "사용자는 로그인 버튼을 클릭하여 로그인할 수 있다."
- ✅ "사용자는 대시보드에서 총 자산 금액을 확인할 수 있다."
- ✅ "사용자는 토큰 목록에서 특정 토큰을 선택하여 상세 정보를 조회할 수 있다."
- ❌ "로그인 기능이 작동한다." (모호함)
- ❌ "대시보드를 본다." (주체가 불명확)
